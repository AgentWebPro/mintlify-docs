Tasks is the archive of every template execution. Each row is one run: when it ran, which template, what inputs, what output.

Use Tasks to:

- Review past work and reuse outputs.
- Re-run a template against different inputs.
- Audit what's been executed (useful for shared workspaces with multiple editors).

## What counts as a task

A task is created any time a **template** runs. This includes:

- Templates run from Agent Mode (when you ask Emma to *"run my weekly competitor brief"*).
- Scheduled templates triggered by a cron.
- Templates triggered via the API or MCP.

Conversational chats without a template are **not** stored here. Those live in your Agent Mode history.

## Finding an execution

1. Open **Tasks** from the sidebar.
2. Filter by template, date range, or status.
3. Click an execution to see inputs, output, and any artifacts produced.

## Re-running a template

From an execution detail page, click **Re-run** to create a new execution with the same inputs. Edit any field before submitting.
